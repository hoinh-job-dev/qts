システム概要

本システムは、Quantaに対して「誰が」「いくら」投資してくれているかをトークン(還元コード)という形で記録することを目的としています。

本システムの利用者は、ユーザー側に「代理店(マーケター)」「購入者(交換希望者)」、サービス運用側に「オペレーター」がいます。

(参考) Quantaとは、CARDANO Labo.が現在開発中の、Blockchain上で全自動の宝くじプラットフォームです。Vietnamの開発拠点で日々企画が進められています。

アクター

代理店本サービスのユーザーで、トークン販売を広めるマーケター。代理店網を広めることと投資家へQuantaを紹介する活動を行う。
個人情報を登録してオペレーターの審査で認証された後に、代理店として活動を開始できる。
交換希望者本サービスのユーザーで、実際に投資を行うトークンの所有者。
個人情報を登録してオペレータの審査で認証された後に、入金された金額に応じたトークンが発行される。後にトークンをQuantaに交換できる。
オペレーター本サービスの運用者。主にKYC審査と注文の進捗管理、お金の管理を行う。

オペレーターの作業

KYC審査

前提 : 本サービスを利用するユーザーには予め個人情報と身分証明書を登録してもらう。KYC/AML対策のためです。

  1. 本システムの画面にてユーザーの個人情報と身分証明書を確認する。
  2. KYC/AML基準に従い、ユーザーを審査する。(例)対象年齢など
  3. 本システムの画面にてユーザの審査結果を登録する。
  4. ユーザーが承認された場合、本システムがユーザーに応じて次のアクションを行う。
    • 代理店へ承認を通知し、本サービスが使えるようになったことを伝える
    • 交換希望者(銀行振込)へ承認を通知し、銀行振込先を伝える
    • 交換希望者(BTC送金)へ承認を通知し、後に送金先のビットコインアドレスを伝えることを伝える

注文の進捗管理

前提 : 交換希望者から投資の申し込み(トークンの注文)があった時に対応する。

  1. 本システムの画面にて入金待ちの注文一覧を確認する。
  2. 入金を確認する。
    • 銀行振込のユーザーの場合、銀行口座の振込履歴を確認する。
    • BTC送金のユーザーの場合、BlockExplorerでそのユーザーに伝えたビットコインアドレスの状態を確認する。
  3. 本システムの画面にて入金された金額を入力する。
  4. 銀行振込の場合はJPY/BTCで換金して、改めて換金結果をシステムに入力する。
  5. BTCの着金が確認できたら(上記"注文の進捗管理"の3.または4.)、本システムがBTC/USDを経由してUSD/QuantaTokenを換算する。
    同時に本システムが代理店へ支払うコミッション金額を計算してシステムに登録する。
    その後、本システムが還元コードを発行してユーザーへメールで通知する。

お金の管理

  1. 銀行振込のJPYの扱いは、
    1. 銀行振込を確認したら本システムの画面から金額を入力する
    2. JPY/BTCの換金レートを調べて、入金されたJPYに相当するBTCを算出する
    3. 算出したBTCを会社のWalletへ送金して、本システムの画面から送金した金額を入力する
  2. BTC送金のBTCの扱いは、
    1. (前提) 注文ごとに受付るビットコインアドレスを発行している
    2. BlockExplorerで送金を確認したら本システムの画面から金額を入力する
  3. コミッションの扱い
    1. 本システムの画面から各代理店へ送金するコミッション金額を確認する
    2. 会社Walletから各代理店へコミッションの金額を送金する
    3. 本システムの画面から送金したコミッションの状態欄に完了を入力する
  4. トークンの扱い
